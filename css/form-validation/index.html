Client-Side Form validation

Note, client-side validation can be bypassed, server-side validation is essential!

For now this only covers built-in validations, not custom validations

Use "required" attribute to make the field required

Good practice is that required fields should always have an asterisk



Minimum and maximum length validation

"minlength" and "maxlength" attributes achieves this

below the minimum gives a comment saying not enough, above it does not allow text anymore

Number ranges for various number inputs

Up and down arrows don't allow beyond min and max

Can type in wrong numbers, but submit doesn't allow and comment tells user

Pattern recognition such as zip codes

"(\d{5}([\-]\d{4})?)" is the paatern for a zip code - "00000-0000"

Placeholder is also important when requiring a certain pattern

This link explains regular expressions

E-mail and url input types automatically validate the inputs

url requires http or https:

email generally requires an @

Styling validation

We use css to style :valid and :invalid pseudo-classes

e.g. input:valid { border-color: green } (this applies everywhere on this sheet)
